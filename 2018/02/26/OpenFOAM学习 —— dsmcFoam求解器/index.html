OpenFOAM学习 —— dsmcFoam求解器

发表于 2018-02-26 | 分类于 OpenFOAM

1. 先来看看顶层求解器dsmcFoam

得益于C++面向对象的编程理念，OpenFOAM的顶层求解器简单明了，顶层求解器的源文件位于application目录下。里面各个变量的命名方式也非常规范，代表的意思一目了然，可以顺藤摸瓜了解一下OpenFOAM的求解算法。

在目录\applications\solvers\discreteMethods\dsmc\dsmcFoam下，包括一个Make文件夹, 和两个源文件CreateFields.H, dsmcFoam.C。 Make文件夹下包含两个文本文件，files和options，内容分别为

files

options

Make/files文件包含了需要进行编译的程序的全路径以及名字（用 EXE = 来指定）

Make/options 用来包含文件头（用 EXE_INC = 来指定）以及链接库（用 EXE_LIBS = 来指定）

源文件CreateFields.H，当在算例目录下的终端窗口运行dsmcInitialise时被调用，用于初始化dsmc的场参数
CreateFields.H

源文件dsmcFoam.C，是一个应用程序，当在算例目录下的终端窗口运行dsmcFoam时被调用。(我们还可以创建自己的应用程序，这篇博文有一个简单的介绍。)

dsmcFoam.C

包含了头文件

  • postProcess.H
  • setRootCase.H
  • createTime.H
  • createMesh.H
  • createFields.H

可以使用文件查找命令find在$FOAM_SRC目录按文件名查找它们，比如这个setRootCase.H文件

这里的$FOAM_SRC是安装好的OpenFOAM源文件的目录。这里输出有两行，其实第二个文件只是第一个文件的链接（对应Windows的快捷方式）。

我们可以看看setRootCase.H和createTime.H这两个文件的内容，

很简短的几行，基本所有的OpenFOAM求解器都要用到这两个文件。

2. 再来看看底层的源程序

这一步，我们需要弄懂dsmcFoam的底层实现，也就是src文件夹里的内容，包括了OpenFOAM所有类的声明和定义文件。

一开始来到这里会让人感到无能为力，OpenFOAM为了维持代码的简介性，使用了大量的继承、派生和模板，而且继承层数之多令人绝望，有的时候为了查到某一个子类的成员函数的定义，要向上找五六层。这时候建议使用OpenFOAM C++ Source Code Guide进行查找，doxygen生成的代码能够减轻迷茫。

dsmc求解器的源码位于目录$FOAM_SRC\lagrangian\DSMC下，包括四个文件夹clouds, Make, parcels, submodels

clouds下包括dsmc使用的cloud模板类和继承类的声明和定义

  • clouds
    • baseClasses
      • DSMCBaseCloud.H
      • DSMCBaseCloud.C
    • derived
      • dsmcCloud.H
    • Templates
      • DSMCCloud.H
      • DSMCCloudI.H
      • DSMCCloud.C
  • Make
    • files
    • options

files创建一个叫DSMC的动态链接库，其中包含了clouds的基类，以及parcels和submodels的派生类。

options包含文件头以及链接库(注意与前文提到的\applications\solvers\discreteMethods\dsmc\dsmcFoam\Make下的options里的内容对比，可以发现后者多了一个链接库，即上面创建的DSMC动态链接库)

parcels下包括dsmc使用的parcel模板类和继承类的声明和定义

  • parcels
    • derived
      • defineDSMCParcel.C
      • dsmcParcel.C
      • dsmcParcel.H
      • makeDSMCParcelBinaryCollisionModels.C

      • makeDSMCParcelWallInteractionModels.C
    • Templates
      • DSMCParcel.C
      • DSMCParcel.H
      • DSMCParcelI.H
      • DSMCParcelIO.C

      • NoBinaryCollision.C
