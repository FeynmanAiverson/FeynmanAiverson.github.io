objectRegistry 类使用虚函数来实现一个通用的框架。 事实上，当程序运行至 runTime.write() 时，下面几个步骤将被执行：

  1. runTime对象调用 regIOobject::write()
  2. regIOobject 调用虚函数 writeObject(), 通常情况下这会调用 objectRegistry::writeObject(), 但是 Time 重载了这一方法，因此实际上是 runTime 对象调用了 runTime::writeObject()
  3. 在调用通常的 objectRegistry::writeObject() 之前，runTime::writeObject() 首先在位于 [caseName]/[timeName]/uniform 路径写一个名为 time 的 IOdictionary 文件
  4. objectRegistry::writeObject() 遍历其所有的子regIOobject，并调用它们各自的的 writeObject(), 如果子对象是一个普通的 regIOobject, 则调用 regIOobject::writeObject()
  5. regIOobject::writeObject() 会打开 OFstream 并调用函数 writeData(), 该函数必须在所有的 regIOobject 中定义。

dsmcFoam —— 1D relaxation to equilibrium

发表于 2018-03-02 | 分类于 OpenFOAM

这里利用dsmcFoam求解一个简单算例，一维理想气体(Ar, $m=66.3\times 10^{-27}kg$)的趋向平衡过程。
考虑边长为 $10^{-6}m$ 的立方体内的Ar气体，气体密度为$\rho=1.78kg/m^3$，温度为$273K$. 所有边界采用周期边界条件，初始速度设置为均方最可几速度(Maxwellian RMS thermal speed)，碰撞模型采用变硬球(VHS)模型。

平衡态气体的三个特殊速度：

  • most probable thermal speed
    $v_m = \sqrt{\frac{2k_B T}{m}}=\sqrt{\frac{2\times 273\times 1.38\times 10^{-23}}{66.3\times 10^{-27}}}=337m/s$
  • average thermal speed
    $\overline v = \sqrt{\frac{8k_B T}{\pi m}}$
  • RMS thermal speed
    $\overline{v^2} = \sqrt{\frac{3k_B T}{m}}=412.9m/s$

分子数密度: $n=\frac{\rho}{m}=2.68\times 10^{25} m^{-3}$

分子数: $2.68\times 10^7$

初始时刻模拟分子数(dsmc particles): $1600$

初始时刻每个模拟分子代表的分子数: $2.68\times 10^7 / 1600 = 1.68\times 10^3$

时间步长设置为 $10^{-11}s$.
对于非定常问题，为了计算稳态，通常采用求系综平均的方法，增加采样数，减少统计误差。对于本问题，$t=5\times 10^{-7}s$ 后模拟分子数目趋于稳定，因此这里选取时间 $t=5\times 10^{-7}s$ 到 $t=1\times 10^{-6}s$ 的所有样本, 这一时间段对应 $5\times 10^4$ 个时间步。

平衡态速度分布为Maxwell分布：
$P(v)=4\pi(\frac{m}{2\pi k_B T})^{3/2}v^2e^{-0.5mv^2/k_BT}$

发表于 2018-02-24 | 分类于 OpenFOAM

OpenFOAM (Open Source Field Operation and Manipulation) is a framework for developing application executables that use packaged functionality contained within a collection of approximately 100 C++ libraries.

OpenFOAM is shipped with approximately 250 pre-built applications that fall into two categories: solvers, that are each designed to solve a specific problem in fluid (or continuum) mechanics; and utilities, that are designed to perform tasks that involve data manipulation.
